# Fraud Detection in Financial Transactions

## Project Overview
This project focuses on detecting fraudulent financial transactions using machine learning techniques.
The dataset contains transaction records where the goal is to identify fraudulent activities among normal transactions.

## Objectives
* Analyze transaction data.
* Identify class imbalance in fraud cases.
* Perform exploratory data analysis (EDA).
* Apply anomaly detection and classification models.
* Evaluate model performance using standard metrics.

## Tools & Technologies
* Python
* Pandas
* NumPy
* Matplotlib
* Seaborn
* Scikit-learn
* SMOTE (Imbalanced-learn)

## Project Workflow
1. Data Loading and Inspection
2. Data Cleaning and Validation
3. Exploratory Data Analysis (EDA)
4. Class Imbalance Analysis
5. Fraud Detection using Isolation Forest
6. Data Balancing with SMOTE
7. Model Training
   * Logistic Regression
   * Random Forest Classifier
8. Model Evaluation

## Key Insights
* Fraudulent transactions represent a very small portion of the dataset.
* Data balancing techniques improve model performance.
* Machine learning models can effectively identify suspicious transactions.

## Results
The project compares different models and evaluates their effectiveness using metrics such as:
* Accuracy
* Precision
* Recall
* Confusion Matrix
